# Cursor AI 활용법

## 1. 개요

---

ChatGPT나 Claude와 같은 단순 대화형 AI의 경우 기존 코드의 흐름을 모른 상태에서 생성해 주는 코드를 일일히 복사하여 ide에 붙혀넣는 번거로움이 존재했다. Cursor AI의 경우 코드의 맥락을 파악할 수 있는 **AI 기능이 완전히 통합된 코드 편집기**이다. 즉, 코딩을 하는 실시간으로 AI와 대화를 하며 코드 수정을 제안받고 적용할 수 있는 코드 편집기라는 뜻이다. 또한 Cursor AI는 VS Code 기반으로 제작되었기 때문에 기존에 이를 사용하던 사용자라면 쉽게 적응할 수 있는 인터페이스로 구성되어 있다.

### 3.2 Cursor 인라인 편집 기능

인라인 편집 기능이란 `Ctrl + K` 단축키를 통해 원하는 코드 영역을 AI를 활용해 수정할 수 있도록 도와주는 기능이다. 먼저 다음과 같이 수정할 부분을 드래그를 통해 지정하고 `Ctrl + K`를 통해 요구사항을 입력한다.

<br>

![alt text](<./image/스크린샷 2025-06-26 150830.png>)

<br>

요구사항을 입력하고 엔터를 누르면 다음과 같이 코드의 수정 제안을 미리보기 형식을 통해 제공해주는 화면을 확인할 수 있다. 이때 `Ctrl + Shift + Y`를 누르면 제안 반영, `Ctrl + N`을 누르면 제안을 거부한다.

<br>

### 3.3 Cursor Chat 기능

말 그대로 자연어를 통해 Cursor AI와 직접 대화할 수 있는 기능이다. 채팅 기능을 제대로 활용하기 위해서는 Cursor에서 `Context`가 가지는 의미가 무엇인지 정확하게 알 필요가 있다. AI에게 지시를 할 때는 "이거 고쳐줘" 라고 단순히 지시하면 원하는 결과를 얻을 수 없다. 반드시 "무엇을 어떻게 고쳐줘" 라는 **상황 맥락**을 함께 제시해야 한다. `Ctrl + i`를 통해 채팅을 열 수 있다. `Context`를 추가하기 위해서는 `@`를 활용하여 필요한 파일, 문서 등을 추가한다.

<br>

![alt text](<./image/스크린샷 2025-06-30 100903.png>)

<br>

Context를 추가해서 제작한 함수의 내용이 마음에 드는 경우 하단에 `Accept all` 버튼을 클릭하고 아닐 경우 `Reject all` 버튼을 클릭할 수 있도록 한다. `Reject all` 버튼을 클릭하면 요청하기 이전 코드로 돌아오는 것을 확인할 수 있다.

<br>

![alt text](<./image/스크린샷 2025-07-15 113623.png>)

<br>

Cursor AI 채팅 기능에는 3가지 모드가 있다.

* **Agent - 자율 실행 모드**: 자율적으로 코드 베이스를 학습하고 프로젝트를 전체적으로 파악 및 수정하는 기능.
* **Ask - 대화형 상담 모드**: 코드에 대한 질문과 답변, 프로젝트의 단계별 계획에 대한 논의를 이루어낼 수 있는 모드. 사이트 링크를 context로 추가하고 대화할 수도 있다.
* **Manual - 정밀 편집 모드**: Agent 모드가 전반적인 프로젝트를 스스로 수정할 수 있는 기능이라면 Manual 모드는 고치고 싶은 특정 파일 또는 부분만을 지정하여 수정할 수 있는 기능이다.

<br>

![alt text](<./image/스크린샷 2025-07-15 114355.png>)

<br>

Context 기능에는 특정 파일과 폴더뿐만 아니라 특정 라이브러리의 공식문서, 코드의 특정 함수, Git 커밋 히스토리 등을 Context로 추가할 수 있는 기능이 있다.

<br>

![alt text](<./image/스크린샷 2025-06-30 101153.png>)

<br>

### 3.4 Cursor Docs Context

단순히 프로젝트 파일이 아니라 AI에게 공식 문서 전체를 학습하도록 할 수 있다. 예를 들어 `Next.js` 프레임워크를 이용해 개발을 진행한다면 `Next.js` 공식문서 전체를 `Docs`로 등록할 수 있다. Cursor로 개발하기 전에 사용할 기술 스택을 `Docs`로 등록해 둔다면 훨씬 편리하게 개발을 진행할 수 있다. 

<br>

![alt text](<./image/스크린샷 2025-07-15 130304.png>)

<br>

`Docs` 추가 방식은 다음과 같다. `Next.js`나 `React`와 같이 널리 알려진 프레임워크 또는 라이브러리의 경우 자동으로 로드가 가능하다.

<br>

![alt text](<./image/스크린샷 2025-07-15 130746.png>)

<br>

하지만 등록되지 않은 문서를 추가하기 위해서는 우측 상단의 **Settings** > **Indexing & Docs** > **Add Docs** 를 이용해 문서를 직접 등록해야 한다.

<br>

![alt text](<./image/스크린샷 2025-07-15 131000.png>)

### 3.5 Cursor 모델 선택

Cursor 설정에는 어떤 AI 모델을 사용할 것인지 선택할 수 있는 Model 탭이 따로 존재한다. GPT, Claude, Gemini 등 다양한 모델을 지원하며 Auto로 설정할 경우 현재 시점에서 최적의 속도와 품질을 보장하는 모델을 자동으로 선정해준다.

<br>

![alt text](<./image/스크린샷 2025-07-15 131637.png>)

<br>

다만 주의할 부분이 있다. `MAX Mode`를 활성화할 경우 AI 모델의 최대 성능을 이끌어내어 기능을 사용할 수 있지만 과금이 크게 부과될 수 있기 때문에 주의해서 사용해야 한다.

<br>

![alt text](<./image/스크린샷 2025-07-15 132028.png>)
